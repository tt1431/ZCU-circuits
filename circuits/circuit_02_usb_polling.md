# 电路 02 — USB 轮询控制电路

> **项目**：ZCU 区域控制单元 | **子系统**：USB 接口管理
> **核心架构**：MOSFET 开关 + NPN 驱动 + 齐纳保护
> **设计日期**：2026

---

## 📌 一句话说清

**用 MOSFET 做电子开关控制 USB 轮询信号**——MCU 通过 NET1 控制 NPN 驱动 MOSFET 栅极，决定 UB_POLLING 输出状态。Zener 做栅极过压保护。

## ⚡ 核心原理

### 信号路径

```
UB_PR（供电）──→ Q2(P-MOS) ──→ UB_POLLING（输出）
                    ↑
              栅极 ← R5(100K上拉) / Q3(下拉到地)
                              ↑
                    NET1 → R7 → Q3(基极)
```

### 工作逻辑

| NET1 | Q3 | Q2 栅极 | Q2 (P-MOS) | UB_POLLING |
|:--|:--|:--|:--|:--|
| **H** | 导通 | ≈0V（拉低） | **导通** | ≈ UB_PR |
| **L** | 截止 | ≈ UB_PR（R5上拉） | **截止** | 高阻/0V |

> 💡 P-MOS 的导通逻辑：栅极比源极低 $|V_{GS(th)}|$ 就导通（BSS84 典型 -1.5V）。Q3 拉低栅极 → P-MOS 导通 → UB_PR 电压传递到 UB_POLLING。

## 🧠 设计深度问答

### Q1：为什么用 MOSFET 而不是三极管？

| | Q2 (P-MOS BSS84) | 如果用 PNP |
|:--|:--|:--|
| 驱动方式 | 电压控制（栅极低就导通），几乎不耗栅极电流 | 电流控制，基极要持续抽电流 |
| 压降 | $R_{DS(on)} \\approx 10\\Omega$，轻载几乎 0V | $V_{CE(sat)} \\approx 0.1V$ |
| 开关速度 | 快 | 略慢（有存储时间） |
| 静态功耗 | 栅极电流 ≈ 0 | 基极电流 $\\approx V/R_B$ |

在这个电路中，UB_POLLING 可能驱动一定负载（USB 检测），MOSFET 的零静态栅极电流和低导通电阻是选它的理由。

### Q2：为什么 Q3 选数字三极管（DTC143Z）？

DTC143ZCAHE3-TP 是**内置偏置电阻的 NPN**（R1=4.7K 内置）：

- 一颗料顶「三极管 + 基极电阻」，少焊一个 0402
- 配合外接 R7=4.7K，总基极电阻 = 4.7K + 4.7K = 9.4K
- NET1=3.3V 时 $I_B \\approx (3.3-0.7)/9.4K \\approx 0.28mA$，足够驱动
- AEC-Q101 车规，-40~125°C

### Q3：D2（15V Zener）有什么用？

保护 MOSFET 栅极。BSS84 的 $V_{GS(max)} = \\pm 20V$。

- 正常工作时 UB_PR ≤ 12V，栅极电压在 0~12V 范围内，Zener 不导通
- 若 UB_PR 异常升高（如负载突变、抛负载），Zener 在 15V 击穿，钳位栅极电压
- 防止 $V_{GS} > 20V$ 击穿栅极氧化层

**15V 的选取**：高于正常工作电压（12V），低于 MOSFET 栅极耐压（20V），留 5V 余量。

### Q4：R5 为什么选 100K？

R5 的作用是在 Q3 截止时把 Q2 栅极上拉到 UB_PR，让 P-MOS 可靠关断。

- 100K 足够大 → 当 Q3 导通时，流过 R5 的电流 $I = 12V/100K = 0.12mA$，几乎不耗电
- 足够小 → 栅极寄生电容（BSS84 典型 ~20pF）的放电时间 $\\tau = 100K \\times 20pF = 2\\mu s$，开关够快
- Q3 截止时，100K 把栅极拉到 UB_PR → $V_{GS} \\approx 0$ → P-MOS 可靠关断

- 负载电流过大？

## ⚡ 补充：唤醒信号调理电路

UB_POLLING 的输出不是直接给 MCU——中间还有一级**信号调理**，用于从采集信号中提取唤醒边沿。

### 原理图

![唤醒信号调理](../images/circuit_02_wakeup.jpg)

### 信号路径

```
采集信号 ── C2(100nF) ──┬── R8(47K) ──→ WKSCR_DIL（MCU 唤醒引脚）
                        │
UB_POLLING ── R9(1.5K) ─┤
                        │
                        ├── R10(47K) ── GND
                        │
                        └── C3(22nF) ── GND
```

### 工作原理

| 器件 | 功能 |
|:--|:--|
| **C2** (100nF) | **交流耦合**——只传递边沿/跳变，阻断直流电平 |
| **R9** (1.5K) | **直流偏置**——从 UB_POLLING 给耦合后的信号提供参考电平 |
| **R8** (47K) | 串联限流，防止瞬态过冲灌入 MCU 引脚 |
| **R10** (47K) + **C3** (22nF) | **RC 低通滤波**，截止频率 $f_c = 1/(2\pi \times 47K \times 22nF) \approx 154$Hz |

### 为什么这样设计？

**采集信号**是原始信号，可能有各种杂波和不确定的电平。不能直接给 MCU。

- **C2 耦合**：只关心「有没有变化」（边沿），不关心「一直是高还是低」（直流）。USB 插入/拔出时产生边沿 → 唤醒 MCU。
- **R9 上拉到 UB_POLLING**：C2 耦合后信号浮空，需要一个直流参考。UB_POLLING 来自电路 02 的 MOSFET 开关——开关闭合时 UB_POLLING=UB_PR，提供干净的偏置。
- **RC 滤波（154Hz）**：滤除高频噪声和振铃，只让真正的插入/拔出事件通过。

### 两级联动

```
电路02（开关）               唤醒调理
─────────────              ─────────────
NET1=H                    采集信号─→C2─→R8─→WKSCR_DIL
  → Q3导通                           ↑
  → Q2导通                     UB_POLLING─→R9（偏置）
  → UB_POLLING=UB_PR            ↓
                            R10+C3→GND（滤波）
```

**完整流程**：
1. MCU 拉高 NET1 → 电路 02 的 MOSFET 导通 → UB_POLLING 建立
2. 采集信号有边沿 → C2 耦合通过 → R8 送到 WKSCR_DIL
3. MCU 唤醒引脚检测到 → MCU 唤醒 → 开始 USB 枚举/通信
